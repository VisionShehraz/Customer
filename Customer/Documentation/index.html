A) Introduction & Basic Configuration - top

GOTAXI Application is Native Android based Application which comes with Admin Panel.

(a) Pre-Requisites :

  • Android SDK
  • Android Studio
  • Java Development Kit (JDK)

  • http://developer.android.com/intl/es/sdk/index.html

  • Open Android Studio > Open an Existing Android Studio Project > Select Your Project build.gradle file > ok
  • Select Project


  • Browse to directory where you place “customer-master” or "driver-master" project and click OK.


    • (d) How to change the package name?:

    1. Now click on setting gear icon and deselect Compact Empty Middle Package.


    2. Now you will see each package folder is broken into parts



    3. Now right click on the first package folder, refactor and rename.




    4. Now a warning will be displayed but you go ahead and click Rename Package. After that enter your domain name for the package name.




    5. Now in the bottom of Android Studio it will display ‘Find Refactoring Preview’. Here click on ‘Do Refactor’



    6. Now open build.gradle (Module: app) in Gradle Scripts. Here change the application id and click Sync Now. Finally it’s done and you have successfully changed the App package name in Android Studio:

    B) Structure and Setting Maps - top

    a) Open http://console.developer.google.com/

    * Select your Project
    * Select Library From the Side Menu
    * Enable the following Google APIs :

      a. Maps SDK fo Android
      b. Maps JavaScript API
      c. Places SDK for Android
      d. Maps Static API
      e. Place API
      f. Geocoding API
      g. Directions API
      h. Distance Matrix API
      i. Geolocation API

    Warning :

    Starting on July 11, 2018 the Google Cloud Platform has implemented rules related to the use of Maps API.
    You must register a Google Cloud Platform account using a credit card or other payment method. Please visit registration at https://console.cloud.google.com/freetrial/

    b) Open https://console.developers.google.com/flows/enableapi?apiid=maps_android_backend&reusekey=true.

    • Select your project
    • Select Google Maps Android Api
    • Generate and copy the Corresponding API KEY
    • Open \GoTaxi\app\src\main\res\values\google_maps_api.xml
    • Change api_key value in GOOGLE_API_KEY from GOOGLE_API_KEY to the copied API KEY.

    • Open File Java \GoTaxi\app\src\main\java\com\gotaxiride\passenger\gmap\GMapDirection.java
    • Please enter API_KEY Google Maps


    C) Get google-services.json for FCM Push Notification Configuration - top

    a) Open https://console.firebase.google.com

    1. Before you do step change the package name, you must do this step for firebase project configuration
    2. Please note : this step is mandatory to generate google-services.json file

    3. Visit Firebase console
    4. Please register or log in first and then select create new project. In the dialog window enter the name of the project that will be created and the contents of the state and then click the create project to continue.

    5. After successfully creating a project on Firebase then the page will be redirected to the dashboard project that has been made.

    6. Then on the dashboard Firebase select Add Firebase to your Android app. In the next dialog window enter the name of the Android application package and SHA1 of fingerprints Certificate (optional).

    7. Next will download the configuration file json google-services.json automatically, which will be added in the directory app in the Android project.

    8. Follow the steps to generate SHA1 Fingerprint
    9. Add the SHA1 Fingerprint to your app in Console of firebase

    10. Download the google_services.json and mopve to project/app folder

    D) Configure File Android Studio - top


    (a) Server Connection

    1. Open File Java \GoTaxi\app\src\main\java\com\gotaxiride\passenger\api\ServiceGenerator.java
    2. Change API_VERSION to match your URL_WEB

    (b) API Key Configuration (Server_Key Cloud Messaging)

    1. Open \GoTaxi\app\src\main\java\net\gotaxi\passenger\config\General.java
    2. Change FCM KEY New Server Key to match public static final String FCM_KEY. (Point 1)

    update documentation : 25 December 2018

    (c) Currency Setting

    Change Currency to match public static final String MONEY. (Point 2)

    (d) Number SOS

    Change Number SOS to match public static final String NUMBER_SOS. (Point 3)

    (e) RTL MODE

    If you use RTL Language e.g : Arabic Language or other, set true (Point 4)

    (f) Setting distance in KM or Miles (Point 5)

    1. Settings in KM You must set the public static final String UNIT_OF_DISTANCE = "Km";
      AND You must set the public static final Float RANGE_VALUE = 1000f ;

    2. Settings MILES You must set the public static final String UNIT_OF_DISTANCE = "Miles";
      AND You must set the public static final Float RANGE_VALUE = 1609f ;

    (g) Setting menu names on Home

    Please change the name of the Home Menu. The Menu Name must be the same on String.xml (point 6)

    E) Generated Signed APK - top

    To sign your app in release mode in Android Studio, follow these steps:

    1. On the menu bar, click Build → Generate Signed APK.
    2. If you already have a keystore, go to step 4.
    3. On the Generate Signed APK Wizard window, click Create new to create a new keystore.
    4. On the New Key Store window, provide the required information, your key should be valid for at least 25 years, so you can sign app updates with the same key through the lifespan of your app.

    5. On the Generate Signed APK Wizard window, select a keystore, a private key, and enter the passwords for both. Then click Next

    6. On the next window, select a destination for the signed APK and Signature Versions.
    7. Select Signature Version : V1 (Jar Signature)and V2 (Full APK Signature)
    8. click Finish and the signed apk will generated.

    Update documentation Feature : 25 December 2018
