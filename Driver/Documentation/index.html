A) Introduction & Basic Configuration - top

GOTAXI Application is Native Android based Application which comes with Admin Panel.

(a) Pre-Requisites :

  • Android SDK
  • Android Studio
  • Java Development Kit (JDK)

    • (b) Basic Configuration:

  • Extract driver_gotaxi.zip to your workspace
  • Open the extracted folder/project from Android Studio
  • Open \GoTaxi\app\build.gradle * Edit the applicationId from "com.gotaxiride.driver" to your project id. This should be uinque so that there shouldn't be any conflict while uploading to Google Play Store.

    • (c) How to change the package name?:

    1. Now click on setting gear icon and deselect Compact Empty Middle Package.


    2. Now you will see each package folder is broken into parts



    3. Now right click on the first package folder, refactor and rename.




    4. Now a warning will be displayed but you go ahead and click Rename Package. After that enter your domain name for the package name.




    5. Now in the bottom of Android Studio it will display ‘Find Refactoring Preview’. Here click on ‘Do Refactor’



    6. Now open build.gradle (Module: app) in Gradle Scripts. Here change the application id and click Sync Now.



    7. Finally it’s done and you have successfully changed the App package name in Android Studio.

    Open Tools Menu in Android Studio, Select Firebase. Assistant Toolbar will be opened.

    1. Select Cloud Messaging from the Firebase Assistant Toolbar
    2. Click Set up Firebase Cloud Messaging
    3. Click On Connect to Firebase
    4. Complete the process described in the toolbar

    B) Structure and Setting Maps - top

    a) Open http://console.developer.google.com/

    * Select your Project
    * Select Library From the Side Menu
    * Enable the following Google APIs :

      a. Maps SDK fo Android
      b. Maps JavaScript API
      c. Places SDK for Android
      d. Maps Static API
      e. Place API
      f. Geocoding API
      g. Directions API
      h. Distance Matrix API
      i. Geolocation API

    b) Open https://console.developers.google.com/flows/enableapi?apiid=maps_android_backend&reusekey=true.

    • Select your project
    • Select Google Maps Android Api
    • Generate and copy the Corresponding API KEY
    • Open \GoTaxi\app\src\main\res\values\google_maps_api.xml
    • Change api_key value in GOOGLE_API_KEY from GOOGLE_API_KEY to the copied API KEY.

    • Open File Java \GoTaxi\app\src\main\java\com\gotaxiride\driver\gmaps\GMapDirection.java
    • Please enter API_KEY Google Maps


    c) Open https://console.firebase.google.com

    1. Select your project
    2. Open https://developers.google.com/android/guides/client-auth
    3. Follow the steps to generate SHA1 Fingerprint
    4. Add the SHA1 Fingerprint to your app in Console of firebase
    5. Download the google_services.json and mopve to project/app folder

    C) Configure File Android Studio - top


    a) Server Connection

    1. Open \GoTaxi Driver\app\src\main\java\com\gotaxi\driver\service\MyConfig.java
    2. Change API_VERSION to match your URL_WEB

    b) API Key Configuration

    1. Open \GoTaxi\app\src\main\java\net\gotaxi\passenger\config\General.java
    2. Change FCM KEY New Server Key to match public static final String FCM_KEY

    c) Change Application Name and Text Inside the Application

    Open \GoTaxi\app\src\main\res\values\strings.xml
    * Change app_name value from GoTaxi, "< string name="app_name">GoTaxi Driver< /string>"
    to < string name="app_name">Your App Name< /string>

    Note : You need to change string tag values on strings.xml file, because this is contain text components which shown inside the application besides content that is retrieved from the admin panel.

    Open Android Studio \GoTaxi\app\src\main\res\values\colors.xml
    Change colorPrimary, colorPrimayDark and colorAccent colours to change the overall theme colors
    Change colour codes wherever is necessary

    D) Generated Signed APK - top

    To sign your app in release mode in Android Studio, follow these steps:

    1. On the menu bar, click Build → Generate Signed APK.
    2. If you already have a keystore, go to step 4.
    3. On the Generate Signed APK Wizard window, click Create new to create a new keystore.
    4. On the New Key Store window, provide the required information, your key should be valid for at least 25 years, so you can sign app updates with the same key through the lifespan of your app.
    5. On the Generate Signed APK Wizard window, select a keystore, a private key, and enter the passwords for both. Then click Next
    6. On the next window, select a destination for the signed APK and Signature Versions.
    7. Select Signature Version : V1 (Jar Signature)and V2 (Full APK Signature)
    8. click Finish and the signed apk will generated.

    E) Application Logo and Graphics - top

    a) Change Application Logo

    1. Open Android Studio → File → New → Image Asset
    2. You can choose 2 types of your icon type, that is Launcher Icon (Adaptive and Legacy) or (Legacy Only)
    3. Recommended to use Launcher Icon (Adaptive and Legacy)
    4. Browse and choose your image file icon (.png) format with square dimension and start customize your icon
    5. Make sure the resource name is "ic_launcher" → Next → Finish

    b) Change Graphics
